# Privacy, Terms, and Legal-Footer Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship substantive counsel-review-ready Privacy and Terms drafts, remove the three deferred legal items (Licenses, Methodology, Do not sell) from the footer and codebase entirely, and add a regression-guard test so they don't accidentally reappear.

**Architecture:** Two new Next.js page rewrites under `app/(legal)/` using the existing `(legal)/layout.tsx` shell. Three page-file deletions. One footer edit. One sitemap edit. One reference-sweep across `app/`, `components/`, `lib/`. Three new vitest files (one guard test + one metadata test per legal page). All changes land as four commits across six phases.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Vitest 4 with jsdom + RTL, Tailwind 4. Path alias `@/` resolves to repo root.

**Spec:** `docs/superpowers/specs/2026-05-17-privacy-terms-and-footer-cleanup-design.md` — the source of truth for content outlines, entity facts, dispute-resolution choice, and acceptance criteria. The plan implements that spec; if anything below contradicts the spec, the spec wins and the plan needs updating.

### Task 2: Write the failing guard test

**Files:**
- Create: `tests/legal-cleanup.test.ts`

- [ ] **Step 1: Write the failing test**

Create `tests/legal-cleanup.test.ts`:

```typescript
import { describe, expect, it } from "vitest";
import { execFileSync } from "node:child_process";
import { existsSync } from "node:fs";
import { resolve } from "node:path";

/* ===========================================================================
   Regression guard: the three deferred legal items (licenses, methodology,
   do-not-sell) were deleted on 2026-05-17. This test fails if any of them
   reappears in production source (under app/, components/, or lib/) or if
   any of the deleted page files comes back.

   Excludes: node_modules, .next, docs/ (the spec discusses these paths in
   prose), tests/ (this test mentions them), git history.

   If you're adding a legal page back, do it deliberately and update or
   delete this test in the same commit.
   =========================================================================== */

const DELETED_PATHS = ["licenses", "methodology", "do-not-sell"] as const;

const DELETED_FILES = [
  "app/(legal)/licenses/page.tsx",
  "app/(legal)/do-not-sell/page.tsx",
  "app/methodology/page.tsx",
] as const;

describe("legal cleanup regression guard", () => {
  it.each(DELETED_PATHS)(
    "production source contains no reference to /%s",
    (deletedPath) => {
      let stdout = "";
      try {
        stdout = execFileSync(
          "grep",
          [
            "-rn",
            "-E",
            `/${deletedPath}`,
            "--include=*.tsx",
            "--include=*.ts",
            "--include=*.json",
            "app",
            "components",
            "lib",
          ],
          { encoding: "utf8", cwd: resolve(__dirname, "..") },
        );
      } catch (err) {
        // grep exits 1 when nothing matches — that's the passing case.
        const e = err as { status?: number; stdout?: string };
        if (e.status === 1) {
          stdout = "";
        } else {
          throw err;
        }
      }
      expect(stdout, `Found stale references to /${deletedPath}:\n${stdout}`).toBe("");
    },
  );

  it.each(DELETED_FILES)("deleted file no longer exists: %s", (file) => {
    expect(existsSync(resolve(__dirname, "..", file))).toBe(false);
  });
});
```
